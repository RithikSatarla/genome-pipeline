# Genome Analysis Pipeline

A Python pipeline that takes a DNA sequence and outputs a complete computational
biology report: genome composition, predicted genes, translated proteins, and
optional homology search via NCBI BLAST.

Built end-to-end with Biopython, pandas, and matplotlib.

![ORF map](results/orf_map.png)

## What it does

Given a genome (FASTA or NCBI accession), the pipeline runs four analysis steps
and writes plots + structured data to `results/`:

1. **Composition stats** — length, GC content, base counts, sliding-window GC
   content, and GC skew (the `(G−C)/(G+C)` signal that flips at a bacterial
   replication origin).
2. **Gene prediction** — finds open reading frames (ORFs) in all 6 reading
   frames on both strands, above a configurable amino-acid length threshold.
3. **Protein analysis** — for every predicted ORF, computes molecular weight,
   isoelectric point, GRAVY hydrophobicity, and instability index using
   Biopython's `ProteinAnalysis`.
4. **Optional BLAST** *(`--blast`)* — sends the longest predicted proteins to
   NCBI's remote BLAST and saves the top homology hits.

## Quick start

```bash
git clone <this-repo>
cd genome-pipeline
pip install -r requirements.txt

# Run on the bundled demo genome (no internet needed):
python pipeline.py --demo

# Or download a real genome from NCBI — e.g., bacteriophage lambda:
python pipeline.py --accession NC_001416.1 --email you@example.com

# Or use your own FASTA:
python pipeline.py --input my_genome.fasta
```

That's it. Outputs land in `results/` as PNGs, a CSV of every predicted ORF, and
a `summary.json`.

## Why these analyses?

- **GC content** varies systematically across genomes and within them — it
  correlates with thermal stability, codon usage, and horizontal gene transfer.
- **GC skew** is asymmetric replication's fingerprint: in many bacteria the
  skew flips sign at the origin of replication, which is how you can locate
  *oriC* without any annotation.
- **ORF prediction** is the basic structural-annotation step before any
  function inference — every gene starts as a candidate ORF.
- **Protein properties (pI, GRAVY)** distinguish soluble cytoplasmic proteins
  from membrane proteins and inform downstream experiments.

## Project structure

```
genome-pipeline/
├── pipeline.py              # CLI entrypoint
├── requirements.txt
├── src/
│   ├── download.py          # NCBI fetch + local FASTA loader
│   ├── stats.py             # GC content, GC skew, base composition
│   ├── orf_finder.py        # 6-frame ORF prediction
│   ├── proteins.py          # Protein property analysis
│   ├── blast_search.py      # Optional remote BLAST
│   └── plots.py             # All matplotlib plots
├── scripts/
│   └── make_demo_genome.py  # Generates the synthetic demo FASTA
├── data/
│   └── demo_genome.fasta    # Bundled 30 kb synthetic genome
└── results/                 # Generated by `python pipeline.py`
```

## A note on the demo genome

`data/demo_genome.fasta` is a **synthetic** 30 kb sequence with planted ORFs.
It exists so the pipeline runs out of the box without needing an internet
connection to NCBI. To analyze a real organism, use `--accession` or
`--input`. Bacteriophage lambda (`NC_001416.1`, ~48 kb, 73 known genes) is a
classic starting point.
